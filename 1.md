# Plan: Microphone Test Mode

## Goal

Add `--microphone=<device>` flag to the `main` binary. When set, it connects to a local
microphone and runs the full STT→LLM→TTS pipeline exactly as if a Plivo call arrived.
Additionally, per every 20ms audio frame, it prints side-by-side VAD numbers from two
independent VAD libraries for latency/accuracy comparison.

## mic.go structure

### MicrophoneProvider

Implements `TelephonyProvider`. Uses two PortAudio streams (input + output).

```go
type MicrophoneProvider struct {
    audioChan    chan TelephonyEvent  // filled by capture goroutine
    playChan     chan []byte          // µ-law bytes to play
    stopOnce     sync.Once
    stopChan     chan struct{}
    inputStream  *portaudio.Stream
    outputStream *portaudio.Stream
}
```

**Input** (`captureLoop`):
- Open PortAudio input stream: 8000 Hz, mono, int16, 160 samples/callback (= 20ms)
- PortAudio callback writes int16 samples into a channel
- Capture goroutine reads int16 samples → convert to PCM bytes → encode µ-law via `g711.EncodeUlaw`
- Wraps in `TelephonyEvent{Type: "audio", Audio: ulawBytes}`
- Sends to `audioChan`

**Output** (`playLoop`):
- Open PortAudio output stream: 8000 Hz, mono, int16
- Play goroutine reads µ-law bytes from `playChan`
- Decodes µ-law → int16 PCM → writes to output stream

**Interface methods**:

```go
func (p *MicrophoneProvider) Handshake() (string, string, error)
    // Returns ("mic-<unix-timestamp>", "inbound", nil)

func (p *MicrophoneProvider) ReadEvent() (TelephonyEvent, error)
    // Blocks on audioChan; returns TelephonyEvent{Type:"stop"} when stopChan is closed

func (p *MicrophoneProvider) SendAudio(audio []byte) error
    // Non-blocking send to playChan (drop if full — avoids deadlock during barge-in)

func (p *MicrophoneProvider) SendClear() error
    // Drain playChan

func (p *MicrophoneProvider) Close() error
    // Calls stopOnce, closes stopChan, stops PortAudio streams
```

### Device selection (newMicrophoneProvider)

```go
func newMicrophoneProvider(deviceName string) (*MicrophoneProvider, error)
```

Parses `deviceName`:
- `"list"` → print all PortAudio device names + indices, return error (caller exits)
- `"default"` or `""` → use `portaudio.DefaultInputDeviceInfo()`
- numeric string (e.g. `"3"`) → use device by index
- anything else → search `DeviceInfo.Name` for case-insensitive substring match (first hit)

### micMain

```go
func micMain(ctx context.Context, settings *Settings, database *sql.DB, device string, sileroModel string) error
```

1. `portaudio.Initialize()`; defer `portaudio.Terminate()`
2. `newMicrophoneProvider(device)` (handles "list" case)
3. `provider.Handshake()` → gets callID
4. Insert CallRecord with status `"answered"` via `databaseCallsInsert`
5. `micProcess(ctx, callID, "inbound", settings, database, provider, sileroModel)`
6. Return nil

### micProcess

Like `callProcess` from `calls.go` but uses `micGoroutineA` instead of `goroutineA`:

```go
func micProcess(
    ctx context.Context,
    callID string,
    direction string,
    settings *Settings,
    database *sql.DB,
    provider TelephonyProvider,
    sileroModel string,
)
```

Steps (mirror callProcess):
1. `newCallSession(now)`
2. Update CallRecord to "ongoing"
3. Setup processCtx + cancel; goroutine to close provider on cancel
4. Send greeting audio
5. Make utteranceChan, bargeInChan, normalStop
6. `go micGoroutineA(processCtx, provider, utteranceChan, bargeInChan, normalStop, session, processCancel, sileroModel)`
7. Pipeline loop (identical to callProcess — select on processCtx.Done / utteranceChan)
8. Finalize: save transcript, update CallRecord

### micGoroutineA (dual VAD goroutine)

Like `goroutineA` from `calls.go` but runs both VADs per frame and prints.

```go
func micGoroutineA(
    ctx context.Context,
    provider TelephonyProvider,
    utteranceChan chan<- []byte,
    bargeInChan chan<- struct{},
    normalStop chan<- struct{},
    session *CallSession,
    cancel context.CancelFunc,
    sileroModel string,
)
```

**VAD init**:
```go
// Existing production VAD (godeps) drives utterance logic
vad, vadMode, _ := vadNew(VADThreshold)

// Comparison VAD #1: maxhawkins/go-webrtcvad
wvad, _ := maxwebrtcvad.New()
wvad.SetMode(2)

// Comparison VAD #2: silero-vad-go
sdet, _ := speech.NewDetector(speech.DetectorConfig{
    ModelPath:  sileroModel,
    SampleRate: 8000,
    Threshold:  0.5,
    MinSilenceDurationMs: 100,
    SpeechPadMs: 30,
})
frameCounter := 0
```

**Per-frame VAD print** (called for every 20ms frame after µ-law → PCM conversion):

```go
func vadPrint(frameN int, pcmBytes []byte, wvad *maxwebrtcvad.VAD, sdet *speech.Detector) {
    // WebRTC result
    webrtcResult, _ := wvad.Process(8000, pcmBytes)
    webrtcVal := 0
    if webrtcResult {
        webrtcVal = 1
    }

    // Silero result: convert PCM bytes → float32 samples, call Detect on this 20ms chunk
    // Silero returns []Segment; if any segment starts/covers this frame → speech=1
    float32Samples := pcmBytesToFloat32(pcmBytes)
    segments, _ := sdet.Detect(float32Samples)
    sileroVal := 0
    if len(segments) > 0 {
        sileroVal = 1
    }

    fmt.Printf("[VAD #%05d] webrtc=%d silero=%d\n", frameN, webrtcVal, sileroVal)
}
```

The existing production `vadProcessPCMBytes(vad, vadMode, pcm)` still drives utterance
accumulation logic (unchanged from goroutineA).

**Full loop** is identical to goroutineA except:
- After µ-law decode, before passing to production VAD: call `vadPrint(frameCounter, pcm, ...)`
- Increment frameCounter per frame
- When utterance emitted: print `[UTTERANCE] emitted (#%d frames)` separator line

## Audio format chain

```
Microphone (int16, 8kHz, mono)
  → pcm bytes ([]byte, 2 bytes/sample)
    → g711.EncodeUlaw(pcm) → µ-law bytes
      → TelephonyEvent{Type:"audio", Audio:ulaw}
        → ReadEvent() → goroutineA → utterance accumulation
          → sttTranscribe() [µ-law → WAV internally]
            → LLM → TTS → µ-law response
              → provider.SendAudio(ulaw)
                → g711.DecodeUlaw(ulaw) → int16 PCM
                  → PortAudio output stream → speaker
```

## Notes / gotchas

- **PortAudio frame size**: request 160 samples/callback at 8kHz = exactly 20ms. This
  matches the VAD frame size used throughout the codebase. No extra buffering needed.

- **int16 ↔ µ-law**: `g711.EncodeUlaw` takes `[]byte` (2 bytes per int16 sample, little-
  endian). `pcmBytesToFloat32` divides by 32768.0 for Silero.

- **Silero model file**: if not found, `speech.NewDetector` returns an error. Print a clear
  message: "silero model not found: download silero_vad.onnx and pass --silero-model=<path>"

- **PortAudio latency**: PortAudio at 8kHz/160 samples has ~20ms input latency. This is
  expected; we're testing the full pipeline latency.

- **No Plivo/Twilio nonce/HMAC**: not needed since we skip the HTTP answer webhook.

- **Settings still needed**: `settings.toml` must be present (STT/LLM/TTS API keys).
  The mic mode loads settings normally.

- **DB**: mic calls are stored in `database.sqlite` like normal calls. Use
  `--action=report` to see them.

- **barge-in during playback**: MicrophoneProvider.SendAudio is non-blocking. Audio
  played while the user is speaking creates overlap; the barge-in cooldown logic in
  micGoroutineA handles this identically to goroutineA.

- **silero-vad-go Detect semantics**: `Detect` processes samples and returns segments
  (start/end times in seconds). For a 20ms chunk (0.02s window), a segment returned
  means speech was detected. Silero VAD has internal state across calls — each 20ms
  chunk call continues the prior state correctly.
